# SPARK Operating System Contract

This document establishes SPARK as an AI-native Media Operating System. It defines the core operating system primitives governing all internal resources.

---

## 1. Operating System Primitives

### A. Executive (User Console)
* **Definition**: The root administrator authority. Defines policies, manages configurations, and acts as the manual approval checkpoint.

### B. Workspace (Tenant Group)
* **Definition**: The root containment boundary grouping active brand definitions and connected accounts.

### C. Brand (Identity Parameters)
* **Definition**: The configuration block declaring content pillars, niche rules, and active voice parameters.

### D. Department (Service Unit)
* **Definition**: A permanent functional operating division (e.g. Research, Creative) implementing a standard capability interface.

### E. Agent (Task Executer)
* **Definition**: A specialized thread context dynamically resolved by the registry to execute an inbound `AgentTask`.

### F. Task (Execution Unit)
* **Definition**: A provider-agnostic message contract encapsulating input data, capability criteria, and execution policies.

### G. Capability (Skill Definition)
* **Definition**: A standardized parameter tag declaring required model capacities (e.g. webSearch, structuredOutput).

### H. Workflow (Plan Specification)
* **Definition**: A static recipe mapping task sequences generated by the Executive Decision Engine.

### I. Pipeline (Active Sequence Run)
* **Definition**: The execution state tracking a workflow from Discovery to Analytics.

### J. Memory (Persistent Guardrail)
* **Definition**: A registry of strategic rules used to guide creative generation prompts.

### K. Learning (Feedback Loop)
* **Definition**: The system process extracting optimization tips from performance analytics.

### L. Policy (Access rules)
* **Definition**: Runtime rules validating task state transitions based on automation modes.

### M. Runtime (Execution Kernel)
* **Definition**: The orchestrator wrapper coordinating task routing, execution adapters, and DB writes.

### N. Provider (LLM API Adapter)
* **Definition**: Standardized interface translating calls to external LLM endpoints.

### O. Repository (Persistence Layer)
* **Definition**: Data access objects isolating Supabase PostgreSQL queries from the runtime.
